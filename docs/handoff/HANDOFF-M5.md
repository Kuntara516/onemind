# 5. Repository Design Philosophy

During M5, an important repository principle was formally adopted.

The quality of the repository takes precedence over the number of documents.

Future documentation shall:

* solve real architectural problems,
* avoid unnecessary duplication,
* remain implementation-oriented,
* preserve technology neutrality,
* support long-term maintainability.

Document identifiers exist to organize knowledge rather than to encourage filling numerical ranges.

# 11. Lessons Learned

Key lessons from M5:

* Enterprise repositories benefit from fewer, higher-value documents.
* Architecture should remain vendor-neutral while acknowledging industry standards.
* AI architecture deserves dedicated design standards rather than extensions of traditional software architecture.
* Operational readiness is as important as implementation correctness.
* Governance and consistency are critical for long-term repository sustainability.
